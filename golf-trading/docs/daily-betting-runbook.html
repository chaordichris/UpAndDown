UpAndDown Daily Betting Runbook

A basic operator checklist for running the golf trading workflow, placing only approved manual bets, and recording every ticket, fill, result, closing line, and note.

Daily Rule

The default outcome is no bet. Only place a bet if the system creates an approved ticket and the actual sportsbook line is still acceptable when you check it.

DataGolf is the model anchor. Do not add gut-feel picks, manual overlays, parlays, or untracked bets.

Set Up

  1. Open a terminal in golf-trading/.
  2. Confirm .env has DATAGOLF_API_KEY.
  3. Pick one paper-trading database and keep using it.

export PAPER_DB=sqlite:////private/tmp/upanddown-paper-review.db

Fast Path: Use The Operator Console

This is the easiest day-to-day workflow. The console is local only. It does not place bets at the books. It records your manual decisions in the database.

PYTHONPATH=. .venv/bin/python scripts/operator_console.py \
  --database-url "$PAPER_DB" \
  --host 127.0.0.1 \
  --port 8765

Open http://127.0.0.1:8765. Leave the terminal running while you work.

Everyday Checklist

Step What To Do What To Record
1. Run live data Fetch DataGolf matchup odds and compute edges for DraftKings and FanDuel. Use the same PAPER_DB. If there are no edges, stop. No action is a valid result.
2. Create tickets Turn candidates into tickets. Review only approved tickets. Ticket ID, book, golfer, market, side, recommended odds, stake, and sleeve.
3. Check the book Open DraftKings or FanDuel manually. Find the exact market and side. Actual odds available now. If the line moved too much or the market is gone, do not force it.
4. Place or skip Place the approved bet manually if the ticket still makes sense. Do not use place-ticket for a skipped or unfilled bet. For placed bets, record actual odds, actual stake, timestamp, book, and notes. For skipped tickets, keep the reason in your external daily notes.
5. After close Record the closing line for each placed bet. Closing American odds from the same book and market.
6. After settlement Record result and attribution. Win, loss, push, void, or dead heat. Then record attribution for the settled bet.
7. Review open actions Check for tickets still unplaced, bets pending settlement, missing CLV, or missing attribution. Resolve each item before using the week as review evidence.

Command Line Workflow

Use these commands if you prefer the terminal or need to repair a specific record.

1. Dry Run First

PYTHONPATH=. .venv/bin/python scripts/run_pipeline.py \
  --tour pga \
  --books draftkings fanduel \
  --dry-run

2. Persist Candidates

PYTHONPATH=. .venv/bin/python scripts/run_pipeline.py \
  --database-url "$PAPER_DB" \
  --tour pga \
  --books draftkings fanduel

3. Create And Review Tickets

PYTHONPATH=. .venv/bin/python scripts/paper_trade.py ticket-candidates \
  --database-url "$PAPER_DB" \
  --total-bankroll 25000

PYTHONPATH=. .venv/bin/python scripts/paper_trade.py list-tickets \
  --database-url "$PAPER_DB" \
  --unplaced

PYTHONPATH=. .venv/bin/python scripts/paper_trade.py export-tickets \
  --database-url "$PAPER_DB" \
  --unplaced \
  --approved-only

4. Record A Placed Bet

PYTHONPATH=. .venv/bin/python scripts/paper_trade.py place-ticket TICKET_ID \
  --database-url "$PAPER_DB" \
  --actual-odds -105 \
  --actual-stake 25.00 \
  --notes "operator-entered pre-tournament paper placement"

5. Record Settlement, CLV, And Attribution

PYTHONPATH=. .venv/bin/python scripts/paper_trade.py settle-bet BET_ID win \
  --database-url "$PAPER_DB"

PYTHONPATH=. .venv/bin/python scripts/paper_trade.py record-clv BET_ID \
  --database-url "$PAPER_DB" \
  --closing-odds -125

PYTHONPATH=. .venv/bin/python scripts/paper_trade.py record-attribution BET_ID \
  --database-url "$PAPER_DB"

6. Check The Work

PYTHONPATH=. .venv/bin/python scripts/paper_trade.py open-actions \
  --database-url "$PAPER_DB"

PYTHONPATH=. .venv/bin/python scripts/paper_trade.py report \
  --database-url "$PAPER_DB"

What To Write In Notes

Keep notes short but specific. Good notes make the audit trail useful later.

Situation Example Note
Normal paper placement DraftKings checked 8:42 AM PT, market matched ticket, paper placement
Line moved FanDuel moved from -105 ticket to -125 actual, skipped
Book rejected or market missing DraftKings market unavailable at placement check, no bet
Promo or boost Boost noted separately; standard line still recorded for CLV

Guardrails

  • Never place a bet that does not have an approved ticket.
  • Never change the recommended stake upward. If you are unsure, size down or skip.
  • Do not delete awkward records. Record what actually happened.
  • Do not count smoke, fixture, or backtest rows as Phase 3 evidence.
  • Paper trading requires at least 4 tournaments and 60 settled paper bets before the Phase 3 gate should be expected to pass.
If the system says no candidates or no approved tickets, the correct daily action is to record nothing and move on. If an approved ticket is not filled because the line moved or the market disappeared, do not fake a placement.
